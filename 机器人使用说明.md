# 机器人使用说明（Telegram）

在私聊或群组中与机器人对话即可使用。支持**发布项目**、**查看列表**、**点赞/点踩投票**；另有独立的**招标/投标**流程（数据与项目投票分开）。**发布者**可修改或删除自己的项目；在 `config.json` 中配置的**管理员**可修改或删除**任意**项目，并可发布招标、截止投标（见 [README.md](./README.md)）。

部署与运行本服务见 [README.md](./README.md)（脚本使用说明）。

## 命令一览

| 命令 | 说明 |
|------|------|
| `/start` 或 `/help` | 显示帮助与命令列表 |
| `/lang cn`、`/lang en`、`/lang ko` | 设置**仅对你生效**的界面语言（韩语为 ISO 639-1：`ko`）；**从未设置过**时默认为**英语**；`/start`、`/help` 中会提示如何切换语言 |
| `/publish 描述 \| 预算` | 发布项目：` \| ` 前为**描述**（必填），后为**预算**（可省略整条 ` \| …`） |
| `/projects` | 列出所有项目：编号、描述、发布者、👍/👎 数量；每条末尾提示用 `/vote 编号` 投票；**仅当**内容超过 Telegram 单条长度上限时才会拆成多条连续消息（带 Part n/m） |
| `/vote 编号` | 针对某一项目显示投票消息，带 **👍 Upvote** / **👎 Downvote** 按钮 |
| `/edit 编号 新描述` 或 `/edit 编号 描述 \| 预算` | **发布者**或管理员可改；不写 ` \| ` 则只改描述、保留原预算 |
| `/delete 编号` | **发布者**或 **config.json 管理员**可删除该项目 |
| `/tender_add 需求正文` | **仅管理员**：发布一条招标需求（公开展示） |
| `/tenders` | 招标列表（状态、需求摘要）；**每条招标下**列出各**投标**摘要（团队、报价）及该投标的 👍/👎 **数量**；投标很多时只列前若干条并提示用 `/tender_view`；过长时拆多条 |
| `/tender_view 编号` | 某条需求全文 + **全部**投标；**每条投标**显示 👍/👎 数量与按钮（针对该条投标，不是整条招标） |
| `/tender_bid 编号 团队说明 \| 报价` | **任何人**可投标；**第一个** `\|` 后为报价，前面为团队说明 |
| `/tender_close 编号` | **仅管理员**：截止该招标（不可再投，仍可查看） |

招标数据保存在服务端 `tenders_data.json`（与 `projects_data.json` 无关）。

示例：

```text
/publish 做一个开源记账小程序 | 预算 5000
/publish 只要描述不要预算
/projects
/vote 1
/edit 1 改成：团队周报收集工具
/edit 2 新说明 | 新预算
/delete 2
/tender_add 需要外包一个官网改版，见附件需求概要
/tenders
/tender_bid 1 我们团队三年官网经验 | 总价 2 万
/tender_view 1
/tender_close 1
```

## 发布与编号

- 每条 `/publish` 会追加一个项目，编号**单调递增**；删除旧项目后，**不会**复用已用过的编号（当前数据已清空，下一条从 **#1** 开始）。
- 使用**第一个**竖线 `|` 分隔描述与预算；无 `|` 时表示不填预算。列表与投票页会显示预算，未填则显示「未填写」类提示。
- 发布时会记录你的用户 ID、显示名称与 `@用户名`（若有），用于「仅发布者可编辑/删除」的判断。

## 投票方式与规则

1. 使用 **`/vote 项目编号`**，机器人会回复一条带按钮的消息。
2. 点击 **👍** 或 **👎** 即投票；**再点同一按钮**可取消该方向的票。
3. 若你已在 👎 再点 👍，会先去掉 👎 再记 👍（两种方向不会同时算在你头上）。

说明：`/projects` 下列表为文字展示；投票需发送 **`/vote 编号`**（列表底部有当前语言下的简要说明）。语言按用户独立保存，互不影响。

## 在群组中使用

- 机器人可在**群组**与**私聊**中使用。
- 若群里发普通消息机器人**没有反应**，多半是 Bot 开启了 **Group Privacy（群组隐私）**：在 [@BotFather](https://t.me/BotFather) → 你的机器人 → **Bot Settings** → **Group Privacy** 设为 **Off**，或只在群里使用**以 `/` 开头的命令**（不依赖读取普通群消息）。

## 权限说明

- **任何人**可以发布项目、查看列表、投票。
- **`/edit`**、**`/delete`**：默认仅**发布者**可用；若你的 Telegram 数字 ID 写在服务端的 **`config.json` → `admin_ids`** 中，则视为**管理员**，可对**任意**项目执行编辑或删除。管理员修改他人项目时，**不会**改写原发布者显示名（仍显示原作者）。
